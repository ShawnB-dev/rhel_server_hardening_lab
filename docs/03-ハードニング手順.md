# ハードニング手順

> **English version:** [03-hardening-steps.md](03-hardening-steps.md)

このドキュメントでは、RHEL 系サーバーに対して実施したハードニング手順を項目ごとに整理しています。詳細な作業内容や検証結果は `02-プロジェクト概要.md` に記載しています。

## SSH 設定の強化
- root ログインの無効化  
- パスワード認証の無効化  
- 鍵認証の強制  
- SSH ディレクトリおよび鍵ファイルのパーミッション強化  
- SSH サービスの再起動と動作確認  

目的は、外部からの攻撃対象を最小化し、強固な認証方式に統一することです。

## ファイアウォール設定（firewalld）
- firewalld の有効化  
- 必要なサービス（例: SSH）のみ許可  
- 不要なポートの閉鎖  
- デフォルトゾーンの確認と調整  

これにより、サーバーのネットワーク露出を最小限に抑えます。

## システムパラメータ（sysctl）の調整
- カーネル情報の露出制限  
- IP スプーフィング対策（rp_filter）  
- ネットワークスタックの安全性向上  
- `/etc/sysctl.conf` の更新と `sysctl -p` による反映  

これらの設定は、低レベルの攻撃に対する耐性を高めます。

## 監査ログ（auditd）の強化
- auditd の有効化  
- カスタム監査ルールの適用  
- 認証、設定変更、権限操作などの重要イベントを記録  
- 監査ログの確認と永続化設定  

監査ログは、インシデント対応やコンプライアンスに不可欠です。

## 不要サービスの確認
- systemctl によるサービス一覧の確認  
- 不要なサービスの停止・無効化  
- 依存関係の確認  

これにより、潜在的な攻撃対象を減らし、リソース効率も向上します。

## まとめ
本ハードニング手順は、RHEL 系サーバーの基本的なセキュリティを強化するための最小限かつ効果的なベースラインです。詳細な作業内容は `02-プロジェクト概要.md` を参照してください。
