# 08 このラボを再現する方法  
RHEL ハードニング環境をゼロから再構築する手順

このドキュメントでは、本プロジェクトで使用した RHEL ハードニングラボ環境を再現するための手順を説明します。VM の準備、初期設定、ハードニングの適用、検証方法までを網羅し、実運用に近い再現性の高いワークフローを提供します。

---

## 1. 仮想マシンの準備

### 新規 VM の作成
- OS: RHEL 8 または RHEL 9（Minimal Install 推奨）
- CPU: 2 vCPU
- メモリ: 2〜4 GB
- ディスク: 20 GB
- ネットワーク: NAT または Bridged（SSH 接続が必要）

### RHEL のインストール
- *Minimal Install* を選択
- *Standard* リポジトリを有効化
- root パスワードを設定

インストール後、コンソールから `root` でログインします。

---

## 2. システムの更新とパッチ適用

ハードニング前にシステムを最新状態に更新します。

```bash
dnf update -y
dnf upgrade -y
```
カーネル更新があった場合は再起動します。

```bash
reboot
```

---
## 3. 非 root の管理ユーザー作成

root 直接ログインを避けるため、管理用ユーザーを作成します。

```bash
useradd adminuser
passwd adminuser
```
wheel グループに追加します。

```bash
usermod -aG wheel adminuser
```
---
## 4. SSH アクセスの設定

SSH 鍵の生成とコピー
ワークステーション側で実行:

```bash
ssh-keygen -t ed25519
ssh-copy-id adminuser@<server-ip>
```

ハードニング適用前はパスワード認証を一時的に有効のままにしても構いません。

---

## 5. リポジトリのクローン

Git をインストール:
```bash
dnf install -y git
```

リポジトリをクローン:

```bash
git clone https://github.com/<your-username>/rhel_server_hardening_lab.git
cd rhel_server_hardening_lab
```
ディレクトリ構成:

```markdown
configs/
scripts/
docs/
screenshots/
```
---

## 6. ハードニングベースラインの適用

ハードニングスクリプトを実行します。

```bash
sudo bash scripts/apply-hardening.sh
```

適用内容:
- SSH 設定のハードニング
- firewalld ルールの適用
- sysctl カーネルパラメータの設定
- パーミッションと所有権の調整

---

## 7. ハードニングの検証

検証スクリプトを実行します。

```bash
sudo bash scripts/verify-hardening.sh
```

確認内容:
- SSH 設定の構文チェック
- firewalld の有効ルール
- カーネルパラメータ
- auditd の状態
- 主要サービスの稼働状況
結果は以下のスクリーンショットと比較できます。

```markdown
docs/07-スクリーンショット.md
```
---
8. Before/After 設定の比較
configs/ ディレクトリには以下が含まれています。
- sshd_config.before / sshd_config.after
- sysctl.before / sysctl.after
差分確認:
```bash
diff -u configs/sshd_config.before configs/sshd_config.after
diff -u configs/sysctl.before configs/sysctl.after
```
どの設定が変更されたかを明確に確認できます。

## 9. ドキュメントの参照

`/docs` フォルダには以下が含まれています。
- ハードニング手順
- 検証方法
- セキュリティ上の理由
- Before/After 比較
- スクリーンショット説明
- 日本語訳ドキュメント

企業内のセキュリティベースライン文書と同様の構成です。

---

## 10. ラボのリセットまたは再構築（任意）

再現を繰り返したい場合:
- VM スナップショットを戻す
または
- VM を削除して再作成
または
- `*.before` ファイルを使用して手動で設定を戻す

面接や学習用に何度でも再実行できます。
