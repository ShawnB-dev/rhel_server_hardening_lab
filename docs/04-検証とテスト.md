# 検証とテスト

> **日本語版:** [05-Before-After比較.md](05-Before-After比較.md)

ハードニング後のシステムが正しく設定され、正常に動作していることを確認するために、各ステップで検証を行います。本ドキュメントでは、実施した主な検証項目をまとめています。

## SSH の検証
- 設定ファイルの構文チェック  

```bash
sshd -t
```

- パスワード認証が無効化されていることを確認  
```bash
grep PasswordAuthentication /etc/ssh/sshd_config
```
- root ログインが無効化されていることを確認  
```bash
grep PermitRootLogin /etc/ssh/sshd_config
```
- 一般ユーザーによる鍵認証ログインをテスト

## ファイアウォールの検証
- firewalld が有効であることを確認  
```bash
systemctl status firewalld
```
- 許可されているサービスの確認  
```bash
firewalld-cmd --list-all
```
- 不要なポートが公開されていないことを確認

## システムパラメータの検証
- sysctl 設定の再読み込みと確認  
```bash
sysctl -p
```
- 重要パラメータの確認  
```bash
sysctl kernel.kptr_restrict sysctl net.ipv4.conf.all.rp_filter
```
## 監査ログの検証
- auditd が稼働していることを確認  
```bash
systemctl status auditd
- 最近の監査ログを確認  
```bash
ausearch -ts recent
```
- カスタム監査ルールの確認  
```bash
auditctl -l
```
## 機能テスト
- 主要サービスが正常に動作していること  
- SSH 接続が期待通りに動作すること  
- ハードニングによる予期せぬ影響がないこと  

検証は、セキュリティ強化とシステムの可用性を両立させるために重要な工程です。
