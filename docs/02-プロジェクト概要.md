# プロジェクト概要

> **English version:** [02-project-overview.md](02-project-overview.md)

このドキュメントは、RHEL 系サーバーに対して実施したセキュリティハードニングの全工程を詳細にまとめたものです。初期状態の確認から設定変更、検証、最終的な構成の比較まで、一連の作業を時系列で記録しています。

本プロジェクトは、実際の企業環境や自宅ラボで行われるハードニング作業を想定し、再現性と透明性を重視しています。各ステップでは、目的、実施内容、理由、検証結果を明確に記載しています。

## ラボ環境
- OS: RHEL 系ディストリビューション（Rocky Linux / AlmaLinux / CentOS Stream など）
- 権限: 一般ユーザー + sudo 権限
- 接続方法: SSH（鍵認証）
- 想定用途: 小規模企業環境、学習環境、個人ラボ

## 実施した主な作業
- SSH 設定の強化  
- firewalld によるネットワーク制御  
- システムパラメータ（sysctl）の調整  
- auditd による監査ログの強化  
- 不要サービスの確認と整理  

## ドキュメント構成
- `01-概要.md`：ハードニングの目的と全体像  
- `02-プロジェクト概要.md`：本ドキュメント（詳細な作業記録）  
- `03-ハードニング手順.md`：実施した設定変更の要点  
- `04-検証とテスト.md`：各ステップの検証方法  
- `05-Before-After比較.md`：設定変更の比較  
- `06-セキュリティ上の理由.md`：各変更の背景と理由  

## 本ドキュメントの目的
- ハードニング作業の再現性を確保する  
- 設定変更の意図を明確にする  
- トラブルシューティングや監査に役立つ記録を残す  
- 学習者や採用担当者に対して、実践的なスキルを示す  

本プロジェクトは、単なる設定変更の羅列ではなく、実際の運用現場で求められる「理由のあるハードニング」を意識して構成されています。
